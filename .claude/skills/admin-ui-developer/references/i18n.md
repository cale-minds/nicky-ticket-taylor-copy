# i18n — Internationalization

## How it works

- `app/i18n.py` uses `contextvars.ContextVar` for per-request locale (async-safe).
- `t("NAMESPACE.KEY")` returns the translation for the current locale, falling back to English, then to the key itself.
- JSON files are loaded with `lru_cache` — the server **must be restarted** after editing any translation file.
- Locale is detected from the `lang` cookie; default: `en`.

## Supported locales (12)

| Code | Language |
|---|---|
| `en` | English |
| `pt-br` | Português |
| `es` | Español |
| `fr` | Français |
| `ar` | العربية (RTL) |
| `da` | Dansk |
| `he` | עברית (RTL) |
| `it` | Italiano |
| `nl` | Nederlands |
| `zh-Hans` | 中文 |
| `ru` | Русский |
| `el` | Ελληνικά |

RTL is applied automatically: `ar` and `he` receive `dir="rtl"` on the `<html>` element.

## Existing namespaces

`NAV`, `COMMON`, `PROFILE`, `DASHBOARD`, `TENANTS`, `ORDERS`, `WEBHOOKS`, `LOGS`, `LANG`

## Adding a new key

1. Add the key to the correct namespace in `app/translations/en.json` (source of truth).
2. Add it to the other 11 JSON files with the corresponding translation.
3. Use `t("NAMESPACE.NEW_KEY")` in the HTML.
4. Restart the server.

## Placeholders

Use `{{variable}}` in the JSON value and pass `t("NS.KEY", variable=value)` in Python:
```json
"METRIC_ACTIVE": "{{count}} active"
```
```python
t("DASHBOARD.METRIC_ACTIVE", count=str(n))
```
